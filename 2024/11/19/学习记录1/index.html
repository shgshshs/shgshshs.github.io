Truth of Word

全选，将内容设置成其他颜色，透明的文字就显示Flag01=SYC{W0rd_H@5

文件本质：

​ 1.原理：

​ word文件的后缀名是.docx，而它的文件本质是一个zip压缩文件

​ 2.方法：

​ 我们可以在压缩文件中将后缀从.docx改为.zip

打开压缩包，在challenge.zip\word\media 路径内找到Flag03.png

FLAG03=StrUCtu3e!}

注意到这个word的后缀名和平常不一样，是docm

.docm格式是Microsoft Word 2007及以上版本支持的一种文件格式，它通常用于储存包含宏和VBA代码的Word文档。

在菜单栏上点开文件，进入选项

点击自定义功能区，勾选开发工具

之后菜单栏上就会出现开发工具选项卡

修改文档属性，勾选解除锁定

打开文档后根据提示启用宏，点击开发工具选项卡，点击Visual Basic

Flag = @_Ama1n9_

拼接得到完整的flag：SYC{W0rd_H@5@_Ama1n9_StrUCtu3e!}

cimbar

cimbar 是一种高密度 2D 条形码格式。数据存储在彩色图块网格中 – 根据选择的图块以及选择的颜色来绘制图块，对位进行编码。对数据应用 Reed Solomon 纠错，以解释视频 -> 数字解码的有损性质。预计错误率低于 1%，并得到纠正。网格上每个图块（位置）有 16 个可能的符号，每个图块编码 4 位。

简单来说就是将文件转化为图标矩阵码，用其他设备扫码进行传输。

该网址网页端就是一个html文件，无网络环境下浏览器打开也能正常使用

编码器将图片转为动画条形码，可以用扫描，Android APP下载

但这题不用扫描，在网上找到图形对应二进制的转换图，直接用CyberChef解码

比较麻烦，就不再输一遍了

该题涉及 PHP 反序列化的知识点，解析常见的数据类型在序列化后的表现。

基本数据类型的序列化结构

类型 示例值 序列化后的结构 解释
字符串 "Alice" s:5:"Alice"; s 表示字符串，5 是字符串的长度，"Alice" 是值
整数 25 i:25; i 表示整数，25 是值。
浮点数 3.14 d:3.14; d 表示浮点数，3.14 是值
布尔值 true / false b:1; / b:0; b 表示布尔值，1 表示 true，0 表示 false。
null null N; N 表示 null 值，无附加数据。

序列化后的数据结构

序列化的核心目标是将复杂的数据结构转换为字符串形式，便于存储或传输。序列化后的字符串包含所有必要信息，以便通过反序列化完整恢复数据结构。以下以 PHP 为例，解析常见的数据类型在序列化后的表现。

1. 基本数据类型的序列化结构

类型 示例值 序列化后的结构 解释
字符串 "Alice" s:5:"Alice"; s 表示字符串，5 是字符串的长度，"Alice" 是值。
整数 25 i:25; i 表示整数，25 是值。
浮点数 3.14 d:3.14; d 表示浮点数，3.14 是值。
布尔值 true / false b:1; / b:0; b 表示布尔值，1 表示 true，0 表示 false。
null null N; N 表示 null 值，无附加数据。

2. 复合数据类型的序列化结构

(1)数组

解释：

  • a:2: 表示数组，包含 2 个元素。
  • {...}: 大括号内是数组的内容，每个元素由键和值组成。
    • 键和值的格式和类型分别对应：
      • s:4:"name"; 是键，表示字符串 "name"。
      • s:5:"Alice"; 是值，表示字符串 "Alice"。
      • s:3:"age"; 是键，表示字符串 "age"。
      • i:25; 是值，表示整数 25。

（2）嵌套数组

解释：

  • a:1: 表示外层数组，包含 1 个元素。
  • 内层数组序列化为 a:2:{s:4:"name";s:5:"Alice";s:3:"age";i:25;}，表示嵌套的数组结构。

(3) 对象

解释：

  • O:4:"User": 表示对象，4 是类名的长度，"User" 是类名。
  • :2: 表示对象有 2 个属性。
  • {...}: 大括号内是属性的键值对。
    • s:4:"name"; 和 s:3:"age"; 是属性名。
    • s:5:"Alice"; 和 i:25; 是属性值。

3. 序列化结构中的特殊标记

标记 含义

i 整数 (integer)
d 浮点数 (double)
b 布尔值 (boolean)
N 空值 (null)
a 数组 (array)
O 对象 (object)
